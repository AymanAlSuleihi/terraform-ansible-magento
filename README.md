# Terraform + Ansible Magento Host

Provision a Magento Open Source 2.4.8-p4 environment on Hetzner Cloud using Terraform for infrastructure lifecycle management and Ansible for host, service, and application configuration.

It delivers a single-server Magento environment with all prerequisites installed and configured, including server configuration, LEMP stack installation, phpMyAdmin, TLS certificate issuance, Magento installation, and patch application.

This environment is intended for quickly provisioning a Magento host for testing, development, and learning purposes. And with further considerations and adjustments, it serves as a solid foundation for single-server production deployment.

As this repo is primarily intended for development and testing purposes, phpMyAdmin is included for convenience. It should be noted that while the included configuration takes some security precautions, exposing database administration publicly is not recommended for production systems and should be disabled in production environments.

## Overview

The repository delivers the following workflow:

1. Terraform provisions a Hetzner Cloud server and applies first-boot SSH hardening.
2. `scripts/render_inventory.sh` converts Terraform outputs into the live Ansible inventory.
3. Ansible configures the operating system, installs the Magento runtime, installs phpMyAdmin, issues TLS certificates, installs Magento, and applies patches for recent vulnerabilities.
4. The playbook returns a post-install summary artifact containing the generated Magento admin URI and other access details.

## Scope

- Single `hcloud_server` on Hetzner Cloud
- Ubuntu 24.04 base image
- SSH hardening with a non-root administrative user
- Fail2Ban, unattended-upgrades, swap, and UFW
- nginx, PHP 8.3, MariaDB 11.4, and OpenSearch 3
- Let's Encrypt certificate issuance via webroot challenge
- Magento Open Source 2.4.8-p4 installation via Composer
- phpMyAdmin behind a custom HTTPS path protected with nginx basic auth
- Magento patch application using `cweagans/composer-patches` and backported security patches for recent CVEs by `hryvinskyi`

## Prerequisites

Before using this repository, ensure the following are available:

- Terraform `>= 1.7`
- Ansible available locally
- A Hetzner Cloud project and API token
- At least one SSH key already registered in the Hetzner Cloud project
- A domain name pointed at the server IP before requesting Let's Encrypt certificates
- Magento Marketplace credentials for `repo.magento.com`

## Authentication

The recommended Terraform authentication mechanism is the `HCLOUD_TOKEN` environment variable:

```bash
export HCLOUD_TOKEN="your-token-here"
```

## Quick Start

1. Create a local Terraform variables file.

   ```bash
   cp terraform/terraform.tfvars.example terraform/terraform.tfvars
   ```

2. Edit `terraform/terraform.tfvars` and set the desired Hetzner server attributes and SSH key references.

   It is recommended to replace example username `magento` with your own secure username.

3. Initialise and apply the Terraform configuration.

   ```bash
   cd terraform
   terraform init
   terraform plan
   terraform apply
   ```

4. Generate the live Ansible inventory from Terraform outputs.

   ```bash
   cd ..
   ./scripts/render_inventory.sh
   ```

   This writes `ansible/inventory.yml`. For documentation or manual reference, a reference file is provided at `ansible/inventory.yml.example`.

5. Create a local Ansible group vars file from the example.

   ```bash
   cp ansible/group_vars/magento_servers.yml.example ansible/group_vars/magento_servers.yml
   ```

6. Populate the required application and secret values in `ansible/group_vars/magento_servers.yml`.

   Required values include:

   - `magento_domain`
   - `magento_letsencrypt_email`
   - `magento_db_password`
   - `magento_repo_public_key`
   - `magento_repo_private_key`
   - `magento_admin_email`
   - `magento_admin_password`
   - `magento_opensearch_initial_admin_password`
   - `magento_phpmyadmin_uri_path`
   - `magento_phpmyadmin_basic_auth_user`
   - `magento_phpmyadmin_basic_auth_password`
   - `magento_phpmyadmin_config_storage_password`

   As with Terraform variables, it is recommended to replace example username values such as `admin` and `magento` with your own secure usernames.
   The phpMyAdmin configuration storage database and control user default to `phpmyadmin` and `pma`; only the control-user password must be supplied unless you need different names.

7. Run the Ansible playbook.

   ```bash
   cd ansible
   ansible-playbook playbooks/site.yml
   ```

8. Inspect the generated install summary for the Magento admin URI, protected phpMyAdmin URL, and other access details.

   Local artifact:

   ```bash
   cat ansible/artifacts/<inventory_hostname>-magento-install-summary.yml
   ```

   Remote artifact (default application root):

   ```bash
   sudo cat /var/www/magento/var/ansible-install-summary.yml
   ```

9. Destroy the infrastructure when no longer needed.

   ```bash
   cd ../terraform
   terraform destroy
   ```

## Configuration

### Terraform

Key Terraform inputs are defined in `terraform/variables.tf`.

Notable inputs:

- `server_name`: Hetzner server name
- `server_type`: Hetzner server type such as `cx23`
- `location`: Hetzner location such as `fsn1`, `nbg1`, or `hel1`
- `image`: Operating system image, default `ubuntu-24.04`
- `ssh_keys`: Existing Hetzner SSH key names or IDs
- `admin_username`: Non-root Linux user created for SSH access
- `enable_ipv6`: Toggle public IPv6 assignment
- `backups`: Toggle Hetzner backups
- `delete_protection`: Protect against accidental deletion or rebuild
- `user_data`: Optional additional cloud-init or shell content appended after the built-in hardening

### Ansible

Repository-local Ansible defaults are defined in `ansible/ansible.cfg`, which points to `ansible/inventory.yml` as the active inventory.

The primary Magento configuration surface is `ansible/group_vars/magento_servers.yml.example`. In addition to required credentials and passwords, useful operational toggles include:

- `magento_deploy_mode`: `default`, `developer`, or `production` (defaults to `developer`)
- `magento_enable_https`: Controls certificate issuance and secure store configuration
- `magento_swap_size_mb`: Swap allocation for the host
- `magento_ufw_enable_ipv6`: IPv6 firewall behavior
- `magento_patches_enabled`: Enables Magento patch application workflow

## Generated Files and Artifacts

This repository generates several environment-specific files during normal operation:

- `terraform/.terraform.lock.hcl`: Terraform provider lock file. Commit this file.
- `ansible/inventory.yml`: Generated inventory derived from Terraform outputs.
- `ansible/artifacts/<inventory_hostname>-magento-install-summary.yml`: Local playbook artifact containing Magento access details.
- `/var/www/magento/var/ansible-install-summary.yml`: Remote install summary written on the server by default.

The install summary includes the resolved admin URI, full admin URL, the protected phpMyAdmin URL and basic auth username, deployment mode, and selected host/application metadata.
